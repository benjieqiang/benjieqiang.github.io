组合索引是将n个列组合成一个索引

其应用场景为：频繁的同时使用n列来进行查询，如：where n1 = ‘admin’ and n2 = 666。

如上创建组合索引之后，存在最左前缀匹配，查询：

  • name and email – 使用索引
  • name – 使用索引
  • email – 不使用索引

注意：对于同时搜索n个条件时，组合索引的性能好于索引合并。

为什么会加速查找？
无索引：从前到后依次查找
有索引：
id       创建额外文件(某种格式存储)
name     创建额外文件(某种格式存储)
email     创建额外文件(某种格式存储)  create index ix_name on userinfo3(email);
name  email 创建额外文件(某种格式存储)            

建立索引：
a. 额外的文件保存特殊的数据结构
b. 查询快；插入更新删除慢
c. 命中索引
select * from userinfo3 where email='asdf';
select * from userinfo3 where email like 'asdf'; 慢 从数据表里找
索引种类：
-    hash索引：创建的表中包括某一列数据转为hash值，内存或者硬盘地址；
        单值快
-    btree索引（innodb用的）

  • or

    • 类型不一致
      如果列是字符串类型，传入条件是必须用引号引起来，不会使用索引

    • !=

    • “>”

    • order by
      select name from mytable order by email desc;

      当根据索引排序时候，选择的映射如果不是索引，则不走索引
      特别的：如果对主键排序，则还是走索引：

      select * from mytable order by nid desc;

    • 组合索引最左前缀
      如果组合索引为：(name,email)
      name and email – 使用索引
      name – 使用索引
      email – 不使用索引

配置MySQL自动记录慢日志

slow_query_log_file = /usr/slow.log 日志文件
log_queries_not_using_indexes = OFF 为使用索引的搜索是否记录
